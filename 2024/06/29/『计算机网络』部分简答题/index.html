『计算机网络』部分简答题

『计算机网络』部分简答题

网络协议三要素

  1. 语法
    数据与控制信息的结构或格式
  2. 语义
    要发出何种控制信息，完成何种动作，回应何种应答
  3. 时序
    事件的实现顺序的详细说明
    对于复杂的计算机网络协议，其结构最好使用层次式的。

物理接口有哪些性质

  1. 机械特性：指明接口的形状、大小、尺寸、引线数目和排列、固定、锁定装置等等
  2. 电气特性：指明接口上的电缆的电压范围
  3. 功能特性：指明接口电缆上每种电压表示何意
  4. 过程特性：对于不同功能的各种可能出现时间的顺序

分组转发的特点

  1. 报文分组，增加头部
  2. 交换机（路由器）存储转发
  3. 在目的主机进行合并

各类数据交换方式的特点以及优缺点 xie

  1. 电路交换：
    1. 端到端的传输，在传输前需要先建立物理通路。 由于通信线路为通信双方用户专用，数据直达传输时延，非常小。
    2. 通信双方物理链路一旦建立，随时可以通信，实时性强。
    3. 双方按发送顺序传输数据，不存在失序的问题
    4. 即适合用于传输模拟信号也适合用于传输
  2. 报文交换：
    1. 报文交换不要为通信双方首先建立一个物理通道，不存在建立时延，用户可以随时进性数据发送
    2. 在报文交换中便于设置检验码和数据重发设施，加之节点可以进性路径选择，就可以在某条传输路径发生问题时，可以重新选择另外一条路径传数据，提高传输的可靠性；
    3. 在存储转发中，容易实现速率匹配，通信效率高且迅速。
  3. 分组交换
    1. 有报文交换的优点，而且将报文分片，路由灵活，网络生存性能好
    2. 后一个分组与前一个分组在网络中的转发是并行的，这种流水线的发送方式有利于提高传输效率。
    3. 减少错误重传的数据量，当分组发送出现错误之后没必要重传一整个的报文，只需要重传分组即可。

以太网发展阶段

  1. 从单个网络 APPANEt 到互联网发展阶段，TCP/IP 协议的初步形成
  2. 建成三级结构的 Internet：分为主干网、地区网和校园网
  3. 形成了多层次的 ISP 结构的 Internet；ISP 首次出现

传输媒介有哪几种，各有什么特点

  1. 双绞线
    双绞线分屏蔽双绞线和无屏蔽双绞线。由两根相互绝缘的导线组成。可以传输模拟信号，也可以传输数字信号，有效带宽达 250kHz，通常距离一般为几道十几公里。导线越粗其通信距离越远。在数字传输时，若传输速率为每秒几兆比特，则传输距离可达几公里。一般用作电话线传输声音信号。虽然双绞线容易受到外部高频电磁波的干扰，误码率高，但因为其价格便宜，且安装方便，既适于点到点连接，又可用于多点连接，故仍被广泛应用。

  2. 同轴电缆
    同轴电缆分基带同轴电缆和宽带同轴电缆，其结构是在一个包有绝缘的实心导线外，再套上一层外面也有一层绝缘的空心圆形导线。由于其高带宽（高达 300~400Hz）、低误码率、性能价格比高，所以用作 LAN 中。同轴电缆的最大传输距离随电缆型号和传输信号的不同而不同，由于易受低频干扰，在使用时多将信号调制在高频载波上。

  3. 光导纤维
    光导纤维以光纤维载体，利用光的全反向原理传播光信号。其优点是直径小、质量轻：传播频带款、通信容量大：抗雷电和电磁干扰性能好，五串音干扰、保密性好、误码率低。但光电接口的价格较昂贵。光纤被广泛用于电信系统铺设主干线。

  4. 无线电技术，分为微波和短波，微波可以理解为你家的无线网和联通电信（他们用的是基站，不是卫星，卫星的延迟极高）技术基础是地面中继。
    短波是依赖电离层的反射技术，延迟大不适用于及时同通信，但是覆盖范围大，适合广播。

ADSL 的中文名称及功能和主要特点

  1. 非对称数字用户线
  2. ADSL 是一种非对称的 DSL 线，所谓非对称就是说，上传的速率较低，下载速率较高。
  3. 该技术为使用数字技术对模拟电话用户进性改造，使它能够承载宽带数字业务。将 0-4Khz 的低频信道留给传统的有线电话使用，将原来没有被利用的 4Khz 的以上的频带留给了用户上网使用。

RIP OSFP BGP 主要特点和原理

UDP 面向报文，TCP 面向字节流

  1. UDP 获得应用层的报文后，不拆分也不合并，直接将数据报封装进 UDP 数据部分，即加上头部之后，直接交付给 IP，UDP 对于 IP 交付的用户数据，去掉头部之后直接交付给应用层。
  2. 不论应用层的数据长度多少，TCP 都要对数据按字节编号。根据网络拥塞程度和用户接收缓存的大小设置发送长度。因此是面向字节流的

端口号的作用

  1. 熟知端口：0-1023：标识常规的服务进程
  2. 等级端口：1024-49151 标记没有熟知端口号的的非常规的服务进程
  3. 短暂端口；49152-65535 提供应用程序的临时接口

简述一下因特网标准制定阶段

  1. 因特网草案–这个阶段还不是 RFC 文档。
  2. 建议标准–这个阶段成为 RFC 文档，
  3. 草案标准
  4. 因特网标准

因特网的两大组成部分

  1. 核心部分
    由路由器联网，负责为边缘部分提供高速 ==远程== 的分组交换。

  2. 边缘部分
    由各主机组成，用户直接进性信息共享和信息处理，低速连入核心网。
